📢 **Релиз AI Engineering Handbook v2.8.0 — «The Prompt Physics»**

AI Engineering Handbook — база знаний по проектированию AI-систем в продакшене: архитектурные решения, инструментарий, методология. В этом релизе три направления: эмпирическое исследование форматов промптов, обновление инструментария для работы с AI-консультантами и автоматизация управления документацией.

🧠 **Форматы промптов: оказалось сложнее**

Когда системный промпт передаётся в LLM, токен-стоимость зависит от формата. Раньше мы знали, что Standard YAML дешевле Pretty JSON, а YAML с блочным стилем `|` — самый дорогой вариант. Теперь выяснилось, что это неполная картина.

Мы сгенерировали YAML из одного и того же JSON-промпта двумя инструментами: **PyYAML** (стандартная Python-библиотека) и **yq** (консольная утилита для YAML, использующая `jq` под капотом). Оба производят валидный, семантически идентичный YAML — но разница в токенах оказывается больше 100 на промпте из 150 строк. PyYAML по умолчанию переносит строки длиннее 80 символов, добавляя символы-разделители без смысловой нагрузки. Для некоторых промптов это переворачивает рейтинг: Standard YAML из PyYAML оказывается дороже Pretty JSON.

Вывод: токен-стоимость — это функция трёх переменных: формат, инструмент генерации, токенизатор модели. Выводы из измерений действительны только если зафиксированы все три.

Отдельный вопрос — почему формат влияет не только на стоимость, но и на качество ответа. Структурные символы (скобки JSON, кавычки, отступы YAML) не несут смысловой нагрузки, но модель всё равно их обрабатывает. Внимание модели — это распределение по всем токенам одновременно: добавление структурных символов смещает это распределение, и на каждый токен с реальным содержимым приходится чуть меньшая доля суммарного внимания. Насколько точно это утверждение на уровне архитектуры трансформера — разобрано в анализе A-26016 в репозитории.

В релизе — три связанных материала с воспроизводимым кодом и измерениями: «Format as Architecture», «Token Economics of Prompt Delivery» и «Appendix: YAML Serializer Variance».

⚙️ **Двухфазный рабочий процесс с консультантами**

В репозитории теперь явно разделены два режима работы с AI:

**Исследование** — `ai_brainstorming_colleague` (v0.2.0): свободное обсуждение, архитектурные «а что если», формулировка вопроса. Без формальных ограничений, без обязательных выводов. Когда направление определено, он подготовит структурированный ответ с предложением перейти к следующему этапу.

**Оценка** — `ai_systems_consultant_hybrid` или `devops_consultant`: строгая рецензия с WRC-оценкой (Weighted Response Confidence — взвешенная метрика по трём осям: эмпирические данные, продакшен-практика, применимость к стеку) и проверкой на SVA-ограничения (Smallest Viable Architecture, ADR-26037).

Разделение режимов предотвращает ситуацию, когда инструмент для исследования выдаёт уверенные архитектурные рекомендации без должного обоснования.

🔧 **Стандарт метаданных документов**

Каждый документ в репозитории — статья, ADR, аналитический материал — теперь имеет машиночитаемый заголовок по единой схеме: тип документа, авторы, теги, версия, дата рождения. Написан валидатор `check_frontmatter.py` (работа в процессе), который проверяет соответствие схеме при каждом коммите. Два новых RFC в статусе proposed: ADR-26044 (Skills as Progressive Disclosure Units) и ADR-26054 (JSON as Governance Config Format).
