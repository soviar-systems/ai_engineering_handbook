**Новый инструмент для валидации форматов ссылок в Jupytext-проектах**

В продолжение темы автоматизации качества документации представляю скрипт `check_link_format.py` — инструмент для проверки приоритета `.ipynb` ссылок в MyST/Jupytext проектах.

💡 **Решаемая проблема**

При использовании Jupytext каждый `.md` файл имеет парный `.ipynb`. Однако MyST рендерит только ноутбуки — ссылки на `.md` файлы вызывают скачивание вместо открытия страницы. Ручной поиск таких ошибок в сотнях файлов — утомительная задача.

⚙️ **Решение: `check_link_format.py`**

Скрипт автоматически находит все `.md` ссылки, для которых существует парный `.ipynb`, и сообщает о необходимости замены:

```
LINK FORMAT ERROR: File 'docs/guide.md:15' links to 'intro.md' but paired .ipynb exists.
  Suggested fix: Change to 'intro.ipynb'
```

🔧 **Новая функция: Автоматическое исправление**

Теперь скрипт умеет не только находить, но и исправлять ошибки:

✅ **Интерактивный режим** (`--fix`): Показывает найденные ошибки и спрашивает подтверждение для каждого файла:
```
File: ai_system/2_model/selection/choosing_model_size.md
  Line 33: /ai_system_layers/.../llm_usage_patterns.md → .ipynb

Fix this file? [y/n/q] (q=quit):
```

✅ **Автоматический режим** (`--fix-all`): Исправляет все ошибки без вопросов — идеально для CI/CD и batch-обработки.

🛠️ **Ключевые возможности**

✅ **Нулевые зависимости**: Только стандартная библиотека Python.

✅ **Разделение ответственности**: Скрипт проверяет только формат ссылок. Проверка существования файлов — задача отдельного `check_broken_links.py`.

✅ **Трехуровневая валидация**: Pre-commit хуки → GitHub Actions → ручной аудит.

✅ **42 автотеста**: Полное покрытие всех сценариев, включая новый функционал автоисправления.

📝 **Примеры использования**

```bash
# Только проверка (CI/pre-commit)
uv run tools/scripts/check_link_format.py

# Интерактивное исправление
uv run tools/scripts/check_link_format.py --fix

# Полная автоматизация
uv run tools/scripts/check_link_format.py --fix-all
```
